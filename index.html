Section 01
01 What is Pulse Code Modulation?

Pulse Code Modulation (PCM) is a method used to digitally represent sampled analog signals. It is the standard form of digital audio in computers, telephony, CD-ROMs, and more. PCM converts a continuous analog signal into a discrete sequence of binary numbers — a process that forms the backbone of all modern digital communication systems.

In PCM, the amplitude of the analog signal is sampled at regular time intervals, and each sample is quantized to the nearest value within a defined range. These quantized values are then encoded as binary numbers (bits) and transmitted or stored digitally.

01
Sampling
The analog signal is measured at uniform time intervals defined by the sampling rate (Hz). Per Nyquist's theorem, the rate must be at least 2× the highest signal frequency.
02
Quantization
Each sample is rounded to the nearest discrete level. The number of levels is determined by bit depth. Arduino's ADC uses 10-bit resolution — giving 1024 quantization levels (0–1023).
03
Encoding
Each quantized value is converted to a binary number. The resulting stream of 0s and 1s is the PCM digital signal, ready to be transmitted, stored, or processed.

Section 02
02 Key Formulas & Concepts

Nyquist-Shannon Sampling Theorem — To faithfully reconstruct a signal, the sampling frequency must be at least twice the maximum frequency of the signal.

fs ≥ 2 × fmax
f_s = sampling frequency  |  f_max = highest frequency in signal

Quantization Levels — The number of discrete levels available depends on the bit depth n of the ADC. Arduino uses 10-bit ADC giving 1024 levels.

L = 2n     →     210 = 1024 levels (Arduino Uno)
L = quantization levels  |  n = number of bits

Step Size / Resolution — The voltage difference between each quantization level.

ΔV = Vref / L     →     5V / 1024 ≈ 4.88 mV per step
V_ref = 5V (Arduino reference)  |  Each ADC step = 4.88 mV

Quantization Error — The difference between actual analog value and its quantized digital representation. It is bounded by ±½ step size.

eq = ± ΔV / 2  ≈  ± 2.44 mV
Section 03
03 Our Project — Hardware Implementation

Our project physically demonstrates the PCM process using an Arduino Uno as the ADC engine. A potentiometer acts as the continuously varying analog input, simulating a real-world analog signal. The Arduino samples this signal using its built-in 10-bit ADC at pin A0.

Component PCM Role Description
Potentiometer Analog Signal Source Produces a continuously varying voltage (0–5V) — represents the raw analog input to be sampled
Arduino Uno (ADC) Sampler + Quantizer Samples pin A0 at ~100Hz, quantizes to 10-bit (0–1023), encodes to binary via analogRead()
LED + Resistor Digital Output / Indicator Brightness proportional to ADC value — visually represents the quantized PCM level
USB Serial PCM Data Transmission Transmits encoded digital values at 9600 baud to PC — the actual PCM bitstream
Python + WebSocket Signal Processing / Display Receives PCM data, reconstructs waveform on browser in real time

Key Insight: When you rotate the potentiometer, the voltage at A0 changes smoothly — this is the continuous analog domain. The moment analogRead(A0) is called, the Arduino freezes that voltage, compares it against 1024 voltage steps internally, and outputs the nearest integer. This single function call performs sampling and quantization simultaneously — both core PCM operations — in microseconds.

The LED brightness provides tangible feedback: it represents the quantized PCM level. A dim LED = low ADC value = low voltage sample. A bright LED = high ADC value = high voltage sample. This directly shows quantization in action with a physical indicator.

The serial transmission at 9600 baud represents the PCM bitstream being transmitted over a channel — analogous to how PCM audio is transmitted in telephone networks or stored on CDs. The Python server acts as the receiver, reconstructing the original waveform from the digital samples — completing the full PCM encode-transmit-decode cycle.

Section 05
05 What We Learned

Through this project, we observed that resolution directly affects signal fidelity. With 10-bit ADC, our voltage resolution is ~4.88mV — sufficiently fine for slow-moving signals like a potentiometer. However, for audio signals (up to 20kHz), we would need a sampling rate exceeding 40kHz — far above our 100Hz demo rate, which illustrates the practical meaning of the Nyquist theorem.

We also understood quantization error not as an abstract formula, but as the visible "staircase" effect in the waveform — where smooth analog motion appears as discrete jumps on screen. The real-time display made this concretely visible in a way that no textbook diagram can replicate.

The project tied together concepts from signal processing, digital electronics, serial communication, and web technology — demonstrating that ADC is not an isolated circuit concept but the foundational bridge between the physical and digital worlds.

Conclusion: PCM is the universal language that converts nature's analog signals into the binary language of computers. Our Arduino circuit, humble as it appears, performs the exact same fundamental operations used in professional audio interfaces, medical sensors, and satellite communication systems — just at a smaller scale.
